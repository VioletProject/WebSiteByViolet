HISTORY

Development

Google Chrome is made up of 25 different library with certain codes such as Mozilla's Netscape Portable Runtime, Network Security Services, NPAPI, Skia Graphics Engine, SQLite, and a number of other projects. The V8 JavaScript virtual machine was set aside because it was the most important project. This project is handled by Danish team that was led by Lars Bak. Implementations that are made for small programs have been designed to be simple, because these small programs do not require high performance. Although, the applications that are used by demanding the full capacity of the browser especially when it comes to DOM model and JavaScript.

WebKit was primary used by Google Chrome as a mechanism to display. This software component is decomposed WebCore component to create the own planned schedule. Blink uses only the "webcam" WebKit components, and we can say that Blink does this because of replacement of other components. This process is executed in order to set up a deployment that enable more efficient work. Google Chrome has been tested with automatic testing of the user interface. This example allows performing of testing process which establishes WebKit's layout. This process with such testing became common and can be said to be testing the user interface automatically. Web developers can stay updated on feature support, follow development on the blog, try out the latest web technologies by downloading Safari Technology to preview and report bugs. Apple's WebKit team proposed a new Community Group to discuss the future of 3D graphics on the Web, and to develop a standard API that exposes modern GPU features including general purpose computation.

Gears is an open source browser extension that was made for Google's browser. Gears are used to add features to developers and it is also used to create web applications that can run offline. Later, Gears has been discontinued by Google, because the functionality of these option become available in HTML5 and it has become usual. Google Gears is currently an early access in release of developers. Google is working with a lot of partners to make Gears widely available, and easy for developers to use. The mission with Gears is to enable more powerful web applications. There were added tons of APIs, enabling everything from offline access to parallel computation. Now that these features have all been adopted by browsers, they are available to more developers than we could have reached with Gears alone.

The Chrome product manager Mike Jazayeri announced that Chrome would remove H.264 video codec support for its HTML5 player. He noted that users should to bring Google Chrome more in line with the currently available open codecs available in the Chromium project. Regardless, Google released a version of Chrome on Windows which added hardware-accelerated H.264 video decoding. Next year, Google has launched a beta version of the device with Android 4.0 system. Chrome is installed on many devices that have built-in Android system and it can be said that Chrome is the default browser for this type of device.
