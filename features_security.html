Security

Chrome periodically takes updates of two blacklist, and that is one for phishing and one for malware, and warns users when they attempt to visit a site flagged as potentially harmfull.

Chrome uses a process - allocation model to sandbox tabs. Sandbox is a security mechanism for separating running programs. Using the principle of least privilege, each tab process cannot interact whit critical memory functions or other tab processes. The Sandbox Team is said to have "taken this existing process boundary and made it into a jail". This enforces a computer security model whereby there are two levels of multilevel security (user and sandbox). The sandbox can only respond to communiction requests initiated by the user. On Linux sandboxing uses the seccomp mode (short for secure computing mode) which is a computer security facility in the Linux kernel.

Since 2008, Chrome has been faulted for not including a master password to prevent casual access to a user's passwords, and their developers said that a master password does not provide real security agaist hackers and they refused to implement one. In February 2014, the Windows version was set to ask user to enter the Windows account pasword before showing saved passowrds.

In January 2015, TorrentFreak reported that using Chrome when connected to the internet using a VPN can be a serious security issue due to the browser's support for WebRTC (Web Real - Time Communication).

On September 12, 2016, it was decided that starting with Chrome 56, users will be warned when they visit non - secure HTTP websites to encourage more sites to make the transition to HTTPS.

Security vulnerabilities

A substantial number of security vulnerabilities in Chrome occure in the Adobe Flash Player. For example, in the 2016 Pwn2Own successful attack on Chrome relied on four security vulnerabilities, wherein the two were in Flash, one was in Chrome and one was in the Windows kernel. In 2016, Google announced that, starting in version 53, plans to phase out Flash Player in Chrome. The first phase is to disable Flash for ads and "background analytics", with the ultimate aim of disabling it completely by the end of the year, except on sites that Google thinks that will not work without it, and then it would be re - enabled with the exception of ads and background analytics on a site - by - site basis.

Malware blocking

Chrome is designed to keep users safer and more secure on the web with built - in malware and phishing protection, auto - updates to make sure that users have all the latest fixes, and more.

Before you visit a site that is suspected of containing malware or phishing, Chrome will show you a warning message.

If you encounter a website suspected of containing phishing or malware, with Safe Browsing technology enabled in Chrome, you will see a warning page like the one shown above.

A phishing attack takes place when someone represents as someon else to trick you, in order to obtain your confidental information, and this is usually achieved through a fake website. On the other hand, malware is software installed on your machine often without your knowledge, and is designed to harm your computer or potentially steal information from your computer. Sandboxing helps prevent malware from installing itself on your computer, and also adds an aditional layer of protection against malicous web pages. Chrome checks regulary your security update, to make sure that it's always kept up - to - date.

Plugins

Up to version 45, Chrome supported plug - ins with Netscape Plugin Application Programming Interface (NPAPI), so that plug - ins, such as Adobe Flash Player, run as unrestricted separate processes outside the browser and can not be sandboxed as tabs are. NPAPI is an application programming interface (API) that alllows plug - ins (browser extensions) to be developed for web browsers. Since 2010, Adobe Flash has been integral to Chrome and does not need be installed separately, and is also kept up to date as part of Chrome's own updates. On August 12, 2009, Google replaced NPAPI with PPAPI (Pepper Plugin API), which is more portable and more secure. On April 14, 2015, Google released Chrome v42, and in that version NPAPI was disabled by default. And in version 45, that was released in September 2015, suport for NPAPI was entirely removed. Now, by default, Chrome asks if you want to run Adobe Flash when you visit a page that has it, which means that you have the opportunity to choose whether you want to use it or not. If you are using a mobile device, Flash won't work on Chrome.
